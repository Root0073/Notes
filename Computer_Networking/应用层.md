应用层的具体内容就是精确定义这些通信规则，这些问题的解决必须通过位于不同主机中的多个应用进程之间的通信和协同工作来完成

应用层的许多协议都是基于客户端服务器方式，客户和服务器都是指通信中所涉及的两个应用进程

# 域名系统DNS
域名系统DNS用于把人们使用的机器名字转换为IP地址

互联网的域名系统DNS被设计为一个联机分布式数据库系统，DNS使大多数名字都在本地进行解析

域名到ip地址的解析是由分布在互联网上的许多域名服务器程序共同完成的，人们也常把运行域名服务器的程序的机器叫域名服务器

解析使用的是解析程序

任何一个连接在互联网上的主机或路由器都有一个可被管理的划分，域还可以划分为子域

## 域名服务器
一个服务器所管辖的范围叫做区，各单位根据具体情况来划分自己管辖范围内的区

每一个区设置响应的权限域名服务器

DNS域名服务器也是按照层次来安排的，每一个域名服务器都只对域名体系中的一部分进行管辖，根据域名服务器所起的作用，可以把域名服务器划分为四种不同的类型：
1. 根域名服务器，由13套设备构成
2. 顶级域名服务器
3. 权限域名服务器
4. 本地域名服务器

DNS域名服务器都要把数据复制到几个域名服务器来保存，其中的一个是主域名服务器，其他的是辅助域名服务器

域名的解析过程：
主机向本地域名服务器的查询一般都采用递归查询
本地域名服务器向根域名服务器的查询通常采用迭代查询

为了提高DNS查询效率并减轻根域名服务器的负荷，在域名服务器中广泛采用了高速缓存的方法

# 文件传输协议
## FTP概述
文件共享协议中的另一大类是联机访问，TCP和UDP

## FTP的基本工作原理
一个FTP服务器进程可同时为多个客户进程提供服务，FTP的服务器进程由两大部分组成：一个主进程负责接收新的请求，另外有若干个从属进程，负责处理单个请求

在进行文件传输时，FTP的客户端和主机之间要建立两个并行的TCP连接：控制连接和数据连接

网络文件系统NFS采用另一种思路：NFS允许应用进程先打开一个远地文件，并能在该文件的一个特定位置上开始读写数据，在网络上传送的只是少量的修改数据

## 简单文件传输协议TFTP
他使用UDP数据报，他所占的内存较小

TFTP发送完一个文件块后就等待对方的确认，确认时应指明确认的块编号，发完数据后在规定时间内收不到确认就要重发数据PDU

每次传输的数据报文中有512字节的数据，但最后一次可不足512字节

# 远程终端协议TELNET
用户用TELNET就可以通过TCP连接注册到远地的另一台主机上，同时也能将远地主机的输出通过TCP返回到用户屏幕

客户软件把用户的击键和命令转换成NVT格式，并交送服务器，服务器软件把收到的数据和命令从NVT格式转换为远地系统所需的格式

# 万维网www

万维网是一个分布式的超媒体系统，它是超文本系统的扩充

## HTTP
HTTP是面向事务的
HTTP使用了面向连接的TCP作为运输层协议，保证了数据的可靠传输

HTTP本身是无连接的，无状态的

HTTP的持续连接有两种工作方式，即流水线方式和非流水线方式

使用流水线方式时，客户访问的所有对象只需花费一个RTT时间

HTTP的请求和响应报文是面向文本的，HTTP/2把所有的报文都划分为许多较小的二进制编码的帧

### 代理服务器
又称为万维网高速缓存
，代理服务器把最近的一些请求和响应都暂存在本地硬盘中，当新请求到达时，若代理服务器发现这个请求与暂存的请求相同，就返回暂存的响应

以代理服务器方式构成的内容分发网络CDN

Cookie相当于一个用户的识别码，带有特定的cookie在http请求中可以让服务器跟踪相同cookie的活动

# 动态主机配置协议DHCP
